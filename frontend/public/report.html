Architectural Strengths

  • Clean Agent Pattern — BaseAgent template with automatic DB tracing, timing, error recording
  • Mock/Real Dual Layer — every external service has a mock implementation, enabling zero-key local dev
  • Full Async — 95%+ backend is async/await, proper connection pooling
  • Token Accounting — per-agent, per-model usage tracking preserved in DB
  • Platform Adaptation — automatic image preprocessing for Douyin/Xiaohongshu/Bilibili resolutions
  • Graceful Fallbacks — LLM failures fall back to template-based prompt generation

Architectural Concerns

  • No Event Streaming — sse-starlette in deps but pipeline progress uses polling (2-3s interval)
  • Single SQLite DB — no WAL mode configured, potential write contention under load
  • Background Tasks — uses asyncio.create_task() directly, no task queue (Celery, ARQ) for reliability
  • No Idempotency — pipeline launch has no dedup protection; double-clicks create duplicate runs
  • Artifact Accumulation — data/generated/ grows indefinitely with no cleanup policy

04 — Backend Issues

B-01: Zero Test Coverage Critical
Entire codebase
No test files exist. Every service, agent, and router is untested. Mock implementations exist but are only used for local dev, not as test doubles. Regression risk on every change is extreme.
Fix: Add pytest + pytest-asyncio. Start with agent pipeline integration tests using mock services.
B-02: No Database Migration Framework Critical
app/database.py — create_all() on startup
Schema changes require dropping the database. No alembic or equivalent migration tooling. Production deployments will lose data on schema evolution.
Fix: Add alembic with async support. Generate initial migration from current models.
B-03: Background Tasks Without Reliability High
app/routers/pipeline.py:62 — asyncio.create_task()
Pipeline runs as in-process asyncio tasks. Server restart = lost pipeline. No task persistence, no retry queue, no dead-letter handling.
Fix: Add ARQ (Redis-based async queue) or Celery for durable task execution.
B-04: No Pipeline Deduplication High
app/routers/pipeline.py:48-64 — launch_pipeline()
Double-clicking "generate" creates two identical pipeline runs. No idempotency key or "pending" status check before launch.
Fix: Accept an idempotency key from frontend; check for existing pending/running runs.
B-05: No Artifact Cleanup Policy High
app/config.py — GENERATED_DIR
Generated audio, video clips, and final videos accumulate in data/generated/ indefinitely. Seedance generates ~10MB per 5s clip; disk usage grows rapidly.
Fix: Add retention policy (e.g., auto-delete artifacts older than N days, or after project deletion).
B-06: Deprecated datetime.utcnow() Medium
app/agents/pipeline.py:65,130 · app/routers/pipeline.py (multiple)
Python 3.12 deprecated datetime.utcnow(). Should use datetime.now(timezone.utc).
Fix: Replace all datetime.utcnow() with datetime.now(timezone.utc).
B-07: SSE Dependency Unused Medium
requirements.txt — sse-starlette==2.1.3
Server-Sent Events library installed but not used. Pipeline progress uses polling at 2-3s intervals. SSE would reduce latency and eliminate wasted requests.
Fix: Implement SSE endpoint for pipeline progress; replace polling in frontend.
B-08: SQLite Write Contention Risk Medium
app/database.py
SQLite with default journal mode. Parallel agent writes (audio + video) may conflict. No WAL mode configured; no connection pool tuning for SQLite specifics.
Fix: Enable WAL mode: PRAGMA journal_mode=WAL on connection. Or migrate to PostgreSQL.
B-09: Subtitle Timing is Naive Medium
app/services/tts_service.py — generate_subtitles()
Splits script by sentence boundaries (。！？) with fixed 3s per segment. Ignores actual TTS audio timing. Result: subtitles drift from audio for varied sentence lengths.
Fix: Use TTS service word-level timestamps (if available) or force-align with audio using Whisper.
B-10: Hardcoded System Prompts Low
app/prompts/system_prompts.py
All LLM system prompts are hardcoded Python strings. No versioning, A/B testing, or user customization.
Fix: Move to configurable prompt templates (DB or file-based) with version tracking.
B-11: No Docker / Deployment Config Low
Project root
No Dockerfile, docker-compose.yml, or CI/CD config. Deployment requires manual setup.
Fix: Add multi-stage Dockerfile (backend + frontend build) and docker-compose for local stack.

05 — Frontend Issues

F-01: Silent API Failures Throughout High
App.tsx:27-28 · AutoModeStudio.tsx (polling) · UsageDashboardPage.tsx
.catch(() => {}) used extensively. Users receive no feedback when: material scan fails, pipeline status poll fails, usage data fails to load. No toast/notification system exists.
Fix: Add global toast notification system (e.g., react-hot-toast). Add axios error interceptor.
F-02: Polling Without Backoff or SSE High
AutoModeStudio.tsx · PipelineProgress.tsx · GenerationPanel.tsx
Fixed 2-3s polling intervals for pipeline status. No exponential backoff on failure. Wastes bandwidth for completed/failed pipelines. No connection health detection.
Fix: Switch to SSE for real-time updates; or implement backoff (2s → 4s → 8s on error).
F-03: No API Timeout or Retry High
api/client.ts
Axios client has no timeout configuration. Long-running requests hang indefinitely. No automatic retry logic for transient failures (network blip, 502/503).
Fix: Set timeout: 30000 on axios instance. Add retry interceptor with 3 attempts for GET requests.
F-04: Unused Dependencies Medium
package.json
@tanstack/react-query installed but never imported (axios used directly). react-router-dom installed but app uses single-page state-driven navigation.
Fix: Remove unused deps to reduce bundle size. Or adopt react-query for server state management.
F-05: No Loading Skeletons Medium
All data-fetching components
Components show blank space while loading. No skeleton loaders or shimmer effects. Perceived performance is poor on slower connections.
Fix: Add skeleton components for material grid, pipeline board, and dashboard.
F-06: TypeScript Strictness Disabled Medium
tsconfig.app.json — noUnusedLocals: false
noUnusedLocals and noUnusedParameters are disabled. Dead code accumulates without compiler warnings.
Fix: Re-enable both flags; clean up unused variables that surface.
F-07: Memory Leaks in Polling Medium
AnalysisPanel.tsx · GenerationPanel.tsx
Some polling intervals store timer reference in local variable (not ref). If component unmounts during polling, interval continues firing.
Fix: Use useRef for all interval references; clear in useEffect cleanup.
F-08: No Accessibility (a11y) Low
All components
No ARIA labels, roles, or keyboard navigation. Screen readers cannot parse the UI. Video/audio elements lack captions. Progress indicators lack aria-live regions.
Fix: Add ARIA attributes incrementally. Start with interactive controls and status regions.
F-09: No Error Boundary Low
App.tsx
No React error boundary. Unhandled component errors crash the entire app with a blank screen.
Fix: Wrap App in an ErrorBoundary component with a fallback "reload" UI.

07 — Product Enhancement Proposals

P-01: Background Music & Audio Mixing High Value

Marketing videos need background music for atmosphere. Current pipeline only has TTS narration.

Proposal: Add BGM library (royalty-free tracks by mood/genre). Video editor agent mixes BGM at -12dB under narration, with fade-in/out. Allow user to select mood (upbeat, calm, cinematic) in pipeline config.

Implementation: Add bgm_mood field to pipeline config → orchestrator selects track → editor mixes via FFmpeg amix filter.

P-02: Video Transition Effects High Value

Current clips are hard-cut concatenated. Marketing videos typically use smooth transitions.

Proposal: Add configurable transition effects between clips: cross-dissolve, fade-black, slide, zoom. Map transitions to style (commercial → dissolve, cinematic → fade-black).

Implementation: FFmpeg xfade filter with configurable duration (0.3-1.0s). Adjust clip timing to account for overlap.

P-03: Watermark & Brand Overlay Medium Value

Marketing videos need brand logos, contact info, or call-to-action overlays.

Proposal: Allow uploading a logo/watermark image per project. Position selector (corner, center-bottom). Optional end-card with contact info.

P-04: Batch Multi-Variant Generation Medium Value

Marketers often need 3-5 variants of the same content for A/B testing on different platforms.

Proposal: "Batch mode" that generates N variants with slight prompt variations, different voice tones, or alternate shot orders. Share same audio but vary visuals.

P-05: Video Template Library Medium Value

New users need inspiration and quick-start options. Templates reduce time-to-first-video.

Proposal: Curated template library: restaurant promo, product unboxing, store tour, event highlight. Each template pre-fills script style, shot structure, and voice tone.

P-06: Cost Estimation Before Generation High Value

Users have no idea how much a pipeline run will cost. Token usage shown after-the-fact is too late.

Proposal: Extend preflight check to estimate costs: N images × Seedance API cost + TTS tokens + LLM tokens. Show estimated cost in CNY/USD before launch. Track cumulative spend per project.

08 — Technical Improvement Roadmap

T-01: SSE Real-Time Pipeline Progress Architecture

Replace polling with Server-Sent Events. sse-starlette is already installed.

# Backend: new endpoint
@router.get("/projects/{project_id}/pipeline/{run_id}/stream")
async def stream_progress(project_id, run_id):
    async def event_generator():
        while True:
            run = await get_run(run_id)
            yield {"data": json.dumps(run_status)}
            if run.status in ("completed", "failed"): break
            await asyncio.sleep(1)
    return EventSourceResponse(event_generator())

Frontend: Use EventSource API to receive live updates. Remove setInterval polling.

T-02: Subtitle Timing via Force-Alignment Quality

Current subtitle timing uses naive sentence splitting (fixed 3s per segment). This causes drift.

Proposal: After TTS generates audio, run Whisper (or similar ASR) on the audio to extract word-level timestamps. Generate SRT from actual speech timing. This guarantees subtitle-audio sync.

# Using faster-whisper for alignment
model = WhisperModel("base")
segments, _ = model.transcribe(audio_path, word_timestamps=True)
for seg in segments:
    for word in seg.words:
        srt_entries.append((word.start, word.end, word.word))
T-03: Database Migration with Alembic Infrastructure 
pip install alembic
alembic init -t async migrations
# Generate migration from current models
alembic revision --autogenerate -m "initial schema"
alembic upgrade head

Enables safe schema evolution without data loss.

T-04: Structured Logging + Tracing Observability

Replace stdlib logging with structured JSON logging (e.g., structlog). Add OpenTelemetry spans for each agent execution. Export to Jaeger/Grafana for debugging.
